# AUTOSAR PSP chain — DBC / FIBEX → .art + netgraph

The vendor bus descriptions (CAN DBC files, FlexRay FIBEX XML) are turned
into the gateway's codec + routing here. **Independent** of the
`.art → manifest → build` pipeline — different inputs, different cadence,
different output tree. All steps are real and idempotent. The generated
`.art` carries an `AUTO-GENERATED` header; never hand-edit — re-run the
importer.

## The chain

```
DBC / FIBEX  ──import-dbc / import-fibex──►  package.art + catalog.json   (per bus)
catalog.json ──gen-autosar-system──►         system.art  (one mega-node per bus)
catalog.json ──gen-psp-netgraph──►           netgraph.json  (PDU → bus address)
```

The canonical PSP lives at `vendor/autosar/<ar>/system/<ar>/<bus>/` and the
emitted `.art` is namespaced under **`system.autosar.<ar>.<bus>`** — the
workspace's `system.*` aggregation tree (the workspace symlink
`system/autosar → vendor/autosar/<ar>/system` keeps the FQN reachable
from `system/system.art`).

The artheia CLI's `--package` flag defaults to `vendor.autosar`, but we
**override it to `system.autosar`** so the PSP slots into the workspace's
`system.*` namespace alongside `system.services.*`, `system.supervisor`,
etc. Pass `--package system.autosar.<ar>[.<bus>]` explicitly on every
import + gen-autosar-system run.

## Commands

Assume `PATH="$PWD/.venv/bin:$PATH"`, `AR=mlbevo_gen2` and `PSP=vendor/autosar/$AR`.

```sh
# CAN — fast (seconds)
artheia import-dbc  --dbc $PSP/config/dbc/*KCAN*.dbc --bus kcan \
    --out $PSP/system/$AR/kcan --package system.autosar.$AR

# FlexRay — slow (minutes on a real FIBEX); --no-validate skips the round-trip
artheia import-fibex --fibex $PSP/config/MLBevo_Gen2_Fx_Cluster_*.xml --bus $AR \
    --out $PSP/system/$AR/fibex --package system.autosar.$AR \
    --no-validate
# KNOWN DRIFT (2026-05): the importer appends --bus to --package, so the
# above emits `package system.autosar.mlbevo_gen2.mlbevo_gen2`, NOT the
# `system.autosar.mlbevo_gen2.fibex` that's currently committed. The
# committed file pre-dates that bus-suffix change in the importer.
# Leave the .art as-is until the importer's suffix policy is rethought
# (track in psp-retirement.md or a follow-on task).

# Aggregate per-bus catalogs into one system.art with one mega-node per bus
artheia gen-autosar-system \
    --catalog $PSP/system/$AR/kcan/catalog.json \
    --catalog $PSP/system/$AR/fibex/catalog.json \
    --out     $PSP/system/$AR/system.art \
    --package system.autosar.$AR
```

A FIBEX-scale import (thousands of PDUs) takes minutes. `--no-validate`
skips the post-import parse round-trip — safe because the catalog schema
is well-tested and the emitter is deterministic.

## What lands where

| file | role |
| --- | --- |
| `<PSP>/system/$AR/<bus>/package.art` | per-bus `.art` package — node + interface stubs |
| `<PSP>/system/$AR/<bus>/catalog.json` | machine-readable form: PDUs, signals, bus addresses |
| `<PSP>/system/$AR/system.art` | aggregate: one mega-node per bus, the gateway sees this |
| `<PSP>/system/$AR/<bus>/netgraph.json` | PDU → bus address LUT (output of `gen-psp-netgraph`) |

The catalog is the source of truth for everything downstream: codec
generation, signal-filter REPL, gateway routing — they all consume
`catalog.json`, not the `.art`.

## Codec & routing generators

Once the catalog is in place, the gateway-side codec + routing is
generated by a separate family of artheia commands:

| command | what it emits |
| --- | --- |
| `gen-can-codec` | proto3 + CAN encoder/decoder from a catalog |
| `gen-fibex-codec` | proto3 + FlexRay decoder/dispatch from a catalog |
| `gen-platform-protos` | unified FlexRay+CAN codec across catalogs |
| `gen-codec-dispatch` | `dispatch_local.c` for `libpsp_local.so` |
| `gen-psp-netgraph` | per-bus `netgraph.json` (PDU → bus address) |
| `gen-psp-registry` | `psp_can_registry.{c,h}` for static registration |
| `gen-gw-types` | `gw_bus_types.h` (PSP-derived bus type IDs) |
| `gen-routing` | per-process routing headers |
| `gen-signal-filter` | walks vendor `.art` for `gateway_route` blocks |
| `signal-filter` | interactive REPL for searching platform signals |

Each is one-shot regen; commit the output. To see the full surface:
`artheia --help`.

## Conventions

- **Workspace package prefix is `system.autosar`** — overriding the
  CLI's `vendor.autosar` default. Always pass `--package
  system.autosar.<ar>[.<bus>]` explicitly so the PSP's FQNs line up with
  the rest of the workspace's `system.*` tree.
- **Generated `.art` is read-only.** Re-run the importer; if the output
  is wrong, the bug is in the importer (or the catalog schema), not in
  the emitted file.
- The PSP is a sibling git repo provisioned by `repo sync` — commit and
  push its regenerated `.art` independently of pero_theia.
