> **DONE (2026-06-04).** Per-node tagged loggers + selectable sinks
> (stdio/null/file/syslog), manifest default file logger, rig-level logger
> config (Process.logger / Rig.logger precedence), and live per-node LogLevel
> push via the supervisor (`tdb loglevel` get-all/get-one/set, with
> GetLogLevelConfig read-back boot⊕override) all landed and exercised. Companion
> `manifest-logging-fc-config-service.md` (the supervisor config service half)

Originally a 4-bullet stub in BACKLOG:

- define logging dst in execution manifest: console/file/syslog
- add endpoint for syslog: host/remote
- add default log level for Node in manifest
- add tags to services Node for log/trace

Expanded into the actionable plan below. Two layered concerns:

1. **AUTOSAR log-context tags** — every Adaptive Platform FC has a
   3-letter context ID (`#COM`, `#SMX`, `#UCM`, etc). Today our
   logger writes `[INFO] message`; we want `[INFO][#SMX] message`
   so a grep across multiple FC logs filters cleanly.

2. **Per-process LogLevel** — the operator (via supervisor-GUI or
   supdbg) needs to dial each child's verbosity up/down. Default
   level comes from rig.py → executor.json → child env. Live
   change without restart is a follow-up.

The hash prefix is AUTOSAR-spec literal. We store the bare 3-letter
code in .art and the runtime renders `[#TAG]`.

## Plan

### Step 1 — `tag` attribute on `node atomic` in .art

Grammar extension:

```
node atomic SmDaemon {
    tipc type=0x8001000D instance=0
    tag = "SMX"
    ports { ... }
}
```

textX grammar: `NodeDecl` gains optional `tag=STRING`. LSP completion
updated.

**Fallback**: if a NodeDecl has no `tag = "..."`, gen-app uses the
node name itself (e.g. `SmDaemon`) as the context. So a fresh
`gen-app` invocation against a tag-less .art still produces a
working Log.hh — operator gets `[INFO][SmDaemon] message` instead
of `[INFO][#SMX] message`. The AUTOSAR 3-letter code is a polish,
not a precondition.

Backfill `tag` in 5 in-tree spec .arts:
sm→SMX, com→COM, per→PER, ucm→UCM, log→LOG. log uses
services/log/system/package.art per the #368 layout migration.

### Step 2 — Generate `<fc>/lib/Log.hh` per FC

gen-app emits a thin wrapper alongside `<DaemonClass>.hh`:

```cpp
// AUTO-GENERATED by gen-app — DO NOT EDIT.
// source: services/system/sm/package.art
//
// Per-FC logging context. Wraps platform::runtime::Logger and
// stamps every record with the AUTOSAR context tag (#SMX for sm).

#pragma once

#include "Logger.hh"      // platform/runtime
#include <memory>
#include <string>

namespace ara::sm {

inline constexpr const char* kLogTag = "#SMX";

// Wraps a base Logger so every message gets the FC tag prefix.
class ContextLogger : public platform::runtime::Logger {
public:
    explicit ContextLogger(std::shared_ptr<platform::runtime::Logger> sink)
        : sink_(std::move(sink)) {}

    void log(platform::runtime::LogLevel lvl,
             const std::string& m) noexcept override {
        sink_->log(lvl, std::string("[") + kLogTag + "] " + m);
    }

private:
    std::shared_ptr<platform::runtime::Logger> sink_;
};

inline std::shared_ptr<platform::runtime::Logger>
MakeContextLogger() noexcept {
    return std::make_shared<ContextLogger>(
        platform::runtime::MakeConsoleLogger());
}

}  // namespace ara::sm
```

gen-app's `Daemon.hh.j2` (and statem variant) `#include "Log.hh"` so
handlers + main both see the wrapper without separate boilerplate.

### Step 3 — `Logger::set_level` + main.cc reads THEIA_LOG_LEVEL

`platform::runtime::Logger` gains a coarse-grained level filter:

```cpp
class Logger {
public:
    void set_level(LogLevel l) noexcept { level_.store(l); }
    LogLevel level() const noexcept { return level_.load(); }

    void log(LogLevel lvl, const std::string& m) noexcept {
        if (lvl >= level_.load()) do_log(lvl, m);
    }
protected:
    virtual void do_log(LogLevel, const std::string&) noexcept = 0;
private:
    std::atomic<LogLevel> level_{LogLevel::Info};
};
```

The existing virtual `log()` becomes non-virtual; subclasses override
`do_log()`. `trace/debug/info/warn/error` shorthand methods unchanged.

gen-app's `main.cc.j2` reads the env at boot:

```cpp
auto logger = ara::sm::MakeContextLogger();
if (const char* lvl = std::getenv("THEIA_LOG_LEVEL")) {
    logger->set_level(platform::runtime::parse_log_level(lvl));
}
```

`parse_log_level("trace"|"debug"|"info"|"warn"|"error") → LogLevel`,
case-insensitive, defaults to Info on unknown.

### Step 7 — live ConfigureLogLevel via supdbg — **DONE (#385)**

Scoped to the **supdbg** path only (GUI dialog deferred — see Out of
scope). Mirrors the #361 ConfigureTrace machinery end-to-end.

Topology (control flows *to the supervisor*, which owns the child):

```
supdbg log set-level <node> <level>
  → services/com gRPC  SupervisorView.ConfigureLogLevel(LogLevelCall)
  → TIPC ControlRequest  op_kind=11  (configure_log_level)
  → supervisor.apply_log_level(target_node, level)
       · log_levels_[node] = level
       · worker.env["THEIA_LOG_LEVEL"] = level   (survives restart)
       · push_log_level_to_child(node)            (live, no-restart)
           → send_frame_to_tipc_name(.., kTagLogApplyConfig=0x0400,
                                     LogLevelConfig{target_node,level})
```

Re-push on the first heartbeat-after-gap (same trigger as trace),
so a restarted child re-receives its level even before its env is
re-read.

Wire shape (supervisor `package.art`):

```
message LogLevelConfig          { string target_node; string level }
message ConfigureLogLevelRequest { LogLevelConfig config }
// ControlRequest op_kind 11 → configure_log_level
operation ConfigureLogLevel(in req: ConfigureLogLevelRequest)
    returns ControlReply
```

services/com bridge: `SupervisorView.ConfigureLogLevel(LogLevelCall)`
packages op_kind=11 and forwards over the same TipcUplink as the
child mutators. supdbg: `Client.configure_log_level()` +
`log set-level <node> {trace|debug|info|warn|error}` subcommand.

Files: supervisor `package.art` + `runtime.{h,cpp}`; com
`supervisor_bridge.proto` + `src/main.cpp` + `CMakeLists.txt`
(new ConfigureLogLevelRequest/LogLevelConfig protos); supdbg
`client.py` + `cli.py` (+ regenerated `_gen/` stubs).

The FC-side consumer of the live `kTagLogApplyConfig` frame is the
same open gap as #361's trace FC side; until that lands, the env
path guarantees the correct level on the next (re)start.
